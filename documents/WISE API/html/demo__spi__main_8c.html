Detailed Description

Example application demonstrating WISE_SPI master/slave transfer in regular mode.

This demo demonstrates basic SPI master/slave communication using WISE SPI driver in REGULAR mode.

Build-time role selection:

  • Define DEMO_SPI_ROLE to one of:
    • DEMO_SPI_ROLE_MASTER
    • DEMO_SPI_ROLE_SLAVE

Hardware notes:

  • Connect SPI signals between master and slave: CLK, CS, MOSI, MISO
  • Additional pins HOLD/WP are configured if QUAD mode is selected.
  • Pin mapping is defined by spi_pio_cfg.

Functional notes:

  • Transfer is performed in non-blocking mode and waits for WISE_SPI_EVENT_TRANSFER_DONE event.
  • Master sends a command (SPI_MSG_FMT_CMD_EN) followed by payload data.
  • Slave prepares an RX buffer and waits for the transfer completion event.
