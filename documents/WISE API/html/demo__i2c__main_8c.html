Detailed Description

Example application demonstrating WISE_I2C master/slave communication.

This demo demonstrates basic I2C master/slave communication using WISE I2C driver (Core API WISE I2C APIs). The role is selected at build time via DEMO_I2C_ROLE:

  • Master: sends a test pattern to a slave device.
  • Slave : receives data and prints the received buffer after events.

Hardware notes:

  • Connect SCL/SDA between master and slave.
  • External pull-up resistors are required on both lines (typically 4.7kΩ or 2.2kΩ).
  • Pin mapping is defined by i2c_pio_cfg.

Functional notes:

  • Default speed: 100kHz (I2C_SPEED_MODE_100K when I2C_SPEED_MODE == 0)
  • Addressing: 7-bit when I2C_ADDRESSING == 0
  • Non-blocking transfer completion is indicated by WISE_I2C_EVENT_TRANSFER_DONE
  • Default slave address: 0x50 (I2C_SLAVE_ADDR)
